Este trabalho encontra-se em andamento.
O objetivo é experimentar opções de persistência
do Modelo de Referência do openEHR, doravante
apenas MR.

IMPORTANTE. A persistência proposta aqui faz
uso de Java (o que não é uma restrição). Contudo,
API assume a implementação do MR baseada na
implementação de referência do openEHR. Em ocasião
futura esta API deverá ser "ajustada" para permitir
a persistência do MR independente da linguagem,
o que exigirá alguma estratégia de serialização,
por exemplo, documento XML ou outra, em vez da atual
implementação Java.
